# 边-云协同 IoT 智能环境监测系统 — 虚拟机部署指南

本文档手把手讲解如何把本项目部署到 **3 台 Ubuntu 虚拟机** 上，实现端（VM1）-边（VM2）-云（VM3）协同。
文末附 **Docker Compose 单机一键部署** 作为备选方案。

---

## 一、总体架构与数据流

```
   VM1 (终端)                VM2 (边缘节点)               VM3 (云平台)
 ┌────────────┐  HTTP POST  ┌─────────────┐  HTTP POST  ┌──────────────┐
 │ 多线程模拟  │ ─────────→ │ 异常检测     │ ─────────→ │ SQLite 存储   │
 │ 传感器数据  │  JSON      │ 定时聚合转发 │  JSON      │ Web 仪表盘     │
 └────────────┘            └─────────────┘             └──────────────┘
   端口: 无                  端口: 5000                  端口: 5000 (浏览器访问)
```

**启动顺序固定为：VM3 → VM2 → VM1**（下游先就绪，上游再发数据）。

---

## 二、准备 3 台虚拟机

### 2.1 规格建议

| 虚拟机 | 角色 | 最低配置 | 操作系统 |
|--------|------|----------|----------|
| VM1 | 终端模拟 | 1 vCPU / 1 GB | Ubuntu 20.04+ |
| VM2 | 边缘节点 | 1 vCPU / 1 GB | Ubuntu 20.04+ |
| VM3 | 云平台 | 1 vCPU / 2 GB | Ubuntu 20.04+ |

> 用 VirtualBox / VMware / 云服务器均可。3 台机器**必须在同一内网且能互相 ping 通**。

### 2.2 配置内网互通（关键）

推荐网络模式：
- **VirtualBox**：每台虚拟机添加一块「仅主机网络 (Host-Only)」或「内部网络 (Internal Network)」网卡；
- **VMware**：使用「NAT」或「仅主机」并保证在同一网段；
- **云服务器**：放在同一 VPC/子网，安全组放行 5000 端口。

在每台虚拟机上查看内网 IP：

```bash
ip addr show        # 或 hostname -I
```

记录三台机器的内网 IP，例如：

| 虚拟机 | 示例内网 IP |
|--------|-------------|
| VM1 | 192.168.56.11 |
| VM2 | 192.168.56.12 |
| VM3 | 192.168.56.13 |

验证互通（在 VM1 上执行）：

```bash
ping -c 3 192.168.56.12   # VM1 → VM2
ping -c 3 192.168.56.13   # VM1 → VM3
```

### 2.3 放行防火墙端口

VM2 和 VM3 都监听 **5000** 端口，需放行：

```bash
# 若启用了 ufw
sudo ufw allow 5000/tcp
sudo ufw reload
sudo ufw status
```

> 云服务器还需在控制台「安全组」中放行 TCP 5000（来源限定为内网网段更安全）。

## 三、每台虚拟机的通用准备

在 **每台** 虚拟机上安装 Python 与 pip，并获取项目代码：

```bash
sudo apt update
sudo apt install -y python3 python3-pip python3-venv git

# 方式一：用 git 克隆（若已上传到仓库）
# git clone <你的仓库地址> cloud-edge-iot

# 方式二：用 scp 把本地项目拷到虚拟机
# 在本地执行：scp -r cloud-edge-iot user@<VM_IP>:~/
```

确认 Python 版本 ≥ 3.10：

```bash
python3 --version
```

> 项目使用了 `str | None` 等语法，**要求 Python 3.10 及以上**。Ubuntu 22.04 自带 3.10，Ubuntu 20.04 自带 3.8，需要 `sudo apt install python3.10` 或升级系统。

## 四、VM3：部署云平台（最先启动）

```bash
cd ~/cloud-edge-iot/vm3-cloud

# 创建并激活虚拟环境（推荐，隔离依赖）
python3 -m venv venv
source venv/bin/activate

pip install -r requirements.txt
```

启动服务（监听 0.0.0.0:5000，对外可访问）：

```bash
python3 cloud_server.py
```

看到如下输出即成功：

```
云端平台启动: 0.0.0.0:5000
数据库: data/iot_data.db
 * Running on all addresses (0.0.0.0)
 * Running on http://<VM3_IP>:5000
```

> 程序会自动创建 `data/` 目录与 `iot_data.db` 数据库文件，无需手动建库。

**浏览器验证**：在任意能访问内网的机器打开 `http://192.168.56.13:5000`（替换为 VM3 实际 IP），应看到仪表盘页面（此时还没有数据，是空的）。

### 可选环境变量（VM3）

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `CLOUD_HOST` | `0.0.0.0` | 监听地址 |
| `CLOUD_PORT` | `5000` | 监听端口 |
| `DATABASE_PATH` | `data/iot_data.db` | SQLite 路径 |

## 五、VM2：部署边缘节点

```bash
cd ~/cloud-edge-iot/vm2-edge
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

**关键：通过环境变量把云端地址指向 VM3 的真实 IP**：

```bash
export CLOUD_URL=http://192.168.56.13:5000     # ← 改成 VM3 的内网 IP
export PM25_THRESHOLD=50                        # PM2.5 异常阈值（可选）
export TEMP_THRESHOLD=38                        # 温度异常阈值（可选）
export AGGREGATE_WINDOW=30                      # 聚合窗口秒数（可选）

python3 edge_server.py
```

看到如下输出即成功：

```
边缘节点启动: 0.0.0.0:5000
异常阈值: PM2.5>50.0, 温度>38.0
聚合窗口: 30s, 云端地址: http://192.168.56.13:5000/api/data
 * Running on all addresses (0.0.0.0)
```

### 可选环境变量（VM2）

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `EDGE_HOST` | `0.0.0.0` | 监听地址 |
| `EDGE_PORT` | `5000` | 监听端口 |
| `CLOUD_URL` | `http://vm3-cloud:5000` | **必须改为 VM3 真实 IP** |
| `PM25_THRESHOLD` | `50` | PM2.5 异常阈值 |
| `TEMP_THRESHOLD` | `38` | 温度异常阈值 |
| `AGGREGATE_WINDOW` | `30` | 正常数据聚合窗口(秒) |

## 六、VM1：部署终端模拟器（最后启动）

```bash
cd ~/cloud-edge-iot/vm1-terminal
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

**关键：把边缘地址指向 VM2 的真实 IP**：

```bash

export DEVICE_COUNT=3                           # 模拟终端数量（多终端加分项）
```

```
...
```

### 可选环境变量（VM1）

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `EDGE_URL` | `http://vm2-edge:5000` | **必须改为 VM2 真实 IP** |

## 七、端到端验证

1. **VM2 日志**应出现 `[接收] 正常/异常` 和每 30 秒一次的 `[聚合转发]`；
2. **VM3 日志**应出现 `[存储] 正常/异常/聚合`；
3. **浏览器**打开 `http://<VM3_IP>:5000`：
   - 顶部统计卡片：在线设备数、数据总量、异常记录数在增长；
   - 折线图：可切换 温度/湿度/PM2.5/CO₂，异常点显示为红色；
   - 右侧设备列表：点击可筛选单个设备；
   - 底部异常事件列表实时刷新。

也可直接用命令行验证 VM3 接口（在 VM3 上）：

```bash
curl http://127.0.0.1:5000/api/stats        # 统计摘要
curl http://127.0.0.1:5000/api/devices      # 设备列表
curl "http://127.0.0.1:5000/api/data/latest?limit=10"   # 最新数据
```

**异常检测专项验证**：临时把 VM2 的阈值调低后重启，可让大部分数据触发异常：

```bash
export PM25_THRESHOLD=20
python3 edge_server.py
```

## 八、让服务常驻后台（可选，推荐用 systemd）

直接 `python3 xxx.py` 会随 SSH 断开而退出。生产化运行可用 systemd。以 VM3 为例：

```bash
sudo tee /etc/systemd/system/iot-cloud.service > /dev/null <<'EOF'
[Unit]
Description=IoT Cloud Platform (VM3)
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/cloud-edge-iot/vm3-cloud
ExecStart=/home/ubuntu/cloud-edge-iot/vm3-cloud/venv/bin/python3 cloud_server.py
Restart=always

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now iot-cloud
sudo systemctl status iot-cloud
journalctl -u iot-cloud -f      # 查看实时日志
```

VM2 / VM1 同理（记得在 `[Service]` 段用 `Environment="CLOUD_URL=http://<VM3_IP>:5000"` 等注入环境变量）。

简易替代方案（无需 systemd）：

```bash
nohup python3 cloud_server.py > cloud.log 2>&1 &      # 后台运行
tail -f cloud.log                                     # 查看日志
```

## 九、常见问题排查

| 现象 | 排查方向 |
|------|----------|
| VM1 一直打印 `发送失败` | VM2 是否已启动？`EDGE_URL` 的 IP 是否正确？VM2 防火墙是否放行 5000？ |
| VM2 打印 `转发云端失败` | VM3 是否已启动？`CLOUD_URL` 是否指向 VM3 真实 IP？ |
| 浏览器打不开仪表盘 | VM3 防火墙/安全组是否放行 5000？是否用 `0.0.0.0` 监听（默认是）？ |
| `ping` 不通 | 检查虚拟机网络模式，确保在同一网段，关闭可能拦截的主机防火墙 |
| `python3: command not found` | `sudo apt install -y python3 python3-pip` |
| 版本报错 / 语法错误 | 确认 `python3 --version` ≥ 3.10 |
| 仪表盘有数据但折线很稀疏 | 正常数据每 `AGGREGATE_WINDOW` 秒才聚合上报一次属正常现象；异常数据会立即上报 |

## 十、备选方案：Docker Compose 单机一键部署

若只有一台机器，可用 Docker 一键拉起全部三个节点（容器间用服务名通信，无需改 IP）：

```bash
cd cloud-edge-iot

# 安装 docker 与 compose 插件（Ubuntu）
sudo apt update && sudo apt install -y docker.io docker-compose-plugin
sudo systemctl enable --now docker

# 一键启动
sudo docker compose up -d --build

# 查看日志
sudo docker compose logs -f

# 访问仪表盘
# http://localhost:5000      （VM3 云平台）
# 边缘节点对外端口映射为 5001（http://localhost:5001）

# 停止并清理
sudo docker compose down
```

容器编排关系见 `docker-compose.yml`：`vm1-terminal → vm2-edge → vm3-cloud`，通过自定义 bridge 网络 `iot-network` 互联，服务名即主机名。
